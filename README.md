# Cadence (Resolve Table)

Локальный трекер исходящих писем контрагентам: тематики, статусы ответов, сроки
ожидания, воронка переходов, задачи, месячный отчёт и хранилище документов.

Приложение полностью **клиентское** — бэкенда нет. Все данные хранятся в браузере
пользователя (`localStorage` + `IndexedDB`), синхронизации между устройствами нет:
каждый браузер ведёт свою копию.

---

## Возможности

- **Письма** — таблица из 12 столбцов (дата отправки, контрагент, контакт,
  email/канал, тематика, тема, статус ответа, дата ответа, срок ожидания в днях,
  ответственный, примечание). Срок ожидания считается автоматически.
- **Стадии воронки** — редактируемый список (добавить/переименовать/переставить/
  удалить), переименование каскадно правит все записи.
- **Воронка переходов** — Sankey по реальной истории смены статусов (включая
  возвраты назад) + диаграмма Ганта.
- **Взаимодействия** — журнал контактов, не являющихся письмами: встречи,
  совещания, эксперименты, звонки и любой свой тип. Дата, контрагент, участники и
  примечание — необязательны; фильтр по типу и поиск.
- **Задачи** — отдельный список с сроками, разделом «Выполнено» и месячным
  отчётом (печать/PDF).
- **Документы** — загрузка/хранение/удаление файлов (`.pdf`, `.docx`, картинки и
  любые другие) с привязкой к письму, компании и этапу. Просмотр по компаниям, по
  этапам и историей загрузок; открытие и скачивание.
- **Экспорт** — кнопка «Экспорт в Excel» выгружает все разделы в один `.xlsx`
  (лист на раздел), а «Скачать файлы» — все документы одним `.zip`. Ничего из
  введённого не теряется.
- **Роли** — `admin` (полный доступ) и `viewer` (только просмотр и скачивание).

> ⚠️ **Авторизация клиентская** — это барьер от случайных глаз, а не криптозащита:
> логины/пароли попадают в собранный JS. Настоящая защита требует серверной
> аутентификации. Учётки задаются в [src/auth.ts](src/auth.ts).

## Где хранятся данные

| Данные | Хранилище | Ключ |
| --- | --- | --- |
| Письма | `localStorage` | `resolve-table-items-v2` |
| Стадии воронки | `localStorage` | `resolve-table-stages-v1` |
| Задачи | `localStorage` | `resolve-table-tasks-v1` |
| Взаимодействия | `localStorage` | `resolve-table-interactions-v1` |
| Сессия входа | `localStorage` | `resolve-table-session-v1` |
| Метаданные документов | `localStorage` | `resolve-table-docs-v1` |
| **Файлы документов (бинарь)** | `IndexedDB` | БД `resolve-table-docs`, store `blobs` |

Файлы лежат в IndexedDB (а не в localStorage), потому что localStorage ограничен
~5 МБ и хранит только строки. Метаданные (имя, тип, размер, компания, этап, дата)
дублируются в localStorage, чтобы списки и фильтры строились мгновенно, без чтения
самих файлов.

## Локальная разработка

Требуется Node.js 18+.

```bash
npm install      # установка зависимостей
npm run dev      # дев-сервер на http://localhost:4173
npm run build    # типизация (tsc) + продакшен-сборка в dist/
npm run preview  # локальный предпросмотр собранной dist/
```

### Учётные записи по умолчанию

| Логин | Пароль | Роль |
| --- | --- | --- |
| `cadmin` | `Cadence!2026` | admin |
| `cview` | `View!2026` | viewer |

Сменить — в [src/auth.ts](src/auth.ts) (пересобрать после правки).

### Ручная проверка «всё грузится и сохраняется»

1. Войти как `cadmin`, добавить письмо → обновить страницу (F5): запись на месте.
2. Открыть 📎 в строке письма → загрузить PDF/картинку → закрыть и открыть снова:
   файл в списке, кнопка «Открыть» показывает его, «Скачать» скачивает.
3. F5 → файл по-прежнему открывается (бинарь в IndexedDB пережил перезагрузку).
4. Вкладка «Документы» → группировки «По компаниям / По этапам / История» и фильтры
   работают.
5. Войти как `cview`: формы и кнопки удаления скрыты, но файлы открываются/качаются.

## Деплой на nginx

Это статический SPA — после сборки получается папка `dist/` с готовыми файлами,
которые nginx раздаёт как статику. Бэкенд и Node на сервере не нужны.

### 1. Собрать

```bash
npm ci
npm run build
```

### 2. Скопировать `dist/` на сервер

```bash
# с локальной машины
scp -r dist/* user@server:/var/www/cadence/
```

или соберите прямо на сервере и положите результат в `/var/www/cadence`.

### 3. Конфиг nginx

`/etc/nginx/sites-available/cadence` (или `conf.d/cadence.conf`):

```nginx
server {
    listen 80;
    server_name cadence.example.com;   # или IP сервера

    root /var/www/cadence;
    index index.html;

    # SPA: любой неизвестный путь отдаём в index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Кэш для хешированных ассетов (имена меняются при каждой сборке)
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    gzip on;
    gzip_types text/css application/javascript application/json image/svg+xml;
    gzip_min_length 1024;
}
```

Активировать и перезагрузить:

```bash
sudo ln -s /etc/nginx/sites-available/cadence /etc/nginx/sites-enabled/
sudo nginx -t          # проверка конфига
sudo systemctl reload nginx
```

### 4. ⚠️ Обязательно: HTTPS или localhost

Приложение использует `crypto.randomUUID()` для генерации id. Этот API доступен
**только в защищённом контексте** — по `https://` или на `localhost`. По голому
`http://<IP>` в части браузеров его нет.

В коде есть запасной генератор id (см. `uid()` в [src/storage.ts](src/storage.ts) и
[src/docs.ts](src/docs.ts)), так что критичного падения не будет, но **для рабочего
стенда настройте HTTPS** — это и безопаснее (пароли входа), и включает полноценный
`crypto`. Простейший путь — Let's Encrypt через certbot:

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d cadence.example.com
```

certbot сам добавит `listen 443 ssl`, сертификаты и редирект с 80 на 443.

### Важно про данные

Данные живут **в браузере каждого пользователя**, а не на сервере. Деплой новой
версии не трогает уже введённые письма/документы у пользователей. Перенос и
резервное копирование — кнопками в шапке:

- **«Экспорт в Excel»** (доступно всем) — один `.xlsx` с листами **Письма /
  Взаимодействия / Задачи / Документы / Стадии**. В лист «Письма» входит и история
  смены статусов одной колонкой. Лист «Документы» — перечень файлов (имя, тип,
  размер, компания, этап, дата), без самих файлов.
- **«Скачать файлы»** (только admin) — все загруженные документы одним `.zip`,
  разложенные по папкам-компаниям. Это дополняет Excel-перечень самими файлами.

Так выгружается всё введённое и ничего не теряется. Импорта обратно в приложение
нет — экспорт служит резервной копией и для чтения/передачи данных.
